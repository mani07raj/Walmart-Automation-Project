Framework Overview & Tech Stack

This is a BDD automation framework built for Walmart.com. It follows a Hybrid Architecture combining BDD (Cucumber), Page Object Model, and Data-Driven Testing. The framework includes realistic interaction timing via HumanUtils to handle Walmart's dynamic page behaviour, overlay dismissal logic, and comprehensive reporting via ExtentReports and Log4j2.

OOP Concepts Implementation

🔒 Encapsulation

All page locators are private static final. Only public business methods are exposed to step definitions. DriverFactory keeps ThreadLocal<WebDriver> private — accessed only via getDriver() / setDriver().

🎭 Abstraction

BasePage is abstract — hides all Selenium + HumanUtils complexity. IBrowser is an interface defining the browser contract. Step definitions only see clean business methods, never raw Selenium.

🧬 Inheritance

All 6 page classes extend BasePage and inherit click(), sendKeys(), getText(), jsClick(), waitUtils, actions — zero code duplication across pages.

🔄 Polymorphism

BrowserFactory implements IBrowser — same createBrowserInstance() method creates Chrome, Edge, or Firefox based on runtime configuration input.

📄 Search.feature — Walmart Product Search @regression  @search 
@regression @search
Feature: Walmart Product Search
  As a Walmart customer
  I want to search for products using the search bar
  So that I can find relevant products quickly

  Background:
    Given I am on the Walmart homepage

  @smoke
  Scenario: Search for iPhone using the search bar
    When I type "iPhone" in the search bar
    And I click on the first search suggestion
    Then the search results page should be displayed
    And the search results should be relevant to "iPhone"

  @regression
  Scenario Outline: Search for multiple products
    When I type "<product>" in the search bar
    And I click on the first search suggestion
    Then the search results page should be displayed
    And the search results should be relevant to "<product>"
    Examples:
      | product    |
      | Samsung    |
      | MacBook    |
      | Headphones |
📄 Cart.feature — Add Product to Cart and Validate @regression  @cart 
@regression @cart
Feature: Add Product to Cart from Departments
  As a Walmart customer
  I want to add a product to my cart via Departments navigation
  So that I can purchase it later

  Background:
    Given I am on the Walmart homepage

  @smoke
  Scenario: Add a Toys product to cart and validate cart details
    When I click on the Departments menu
    And I hover over "Toys & Outdoor Play"
    And I click on "All Toys & Outdoor Play"
    And I select the first product from the listing
    Then I should be on the product details page
    When I click on "Add to cart"
    Then the product should be added to the cart
    And the cart subtotal should be displayed
    And the estimated total should be displayed
    And the cart icon count should be "1"

How to Execute & View Reports

Run All Tests

mvn clean test

Run Specific Tags

mvn clean test -Dcucumber.filter.tags="@smoke"
mvn clean test -Dcucumber.filter.tags="@search"
mvn clean test -Dcucumber.filter.tags="@cart"
mvn clean test -Dcucumber.filter.tags="@regression"

Cross-Browser Execution

mvn clean test -Dbrowser=chrome    # Default
mvn clean test -Dbrowser=edge
mvn clean test -Dbrowser=firefox

Run from IntelliJ (no Maven in PATH)

"C:\Program Files\JetBrains\IntelliJ IDEA 2026.1\plugins\maven\lib\maven3\bin\mvn.cmd" clean test
